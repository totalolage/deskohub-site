## 🏗️ Architecture Principles

Our codebase follows these core principles:

1. **Feature-Based Organization**: Code is organized by business features, not technical layers
2. **Self-Contained Modules**: Each feature contains all its necessary code
3. **Clear Boundaries**: Features expose only what's needed through their index files
4. **Shared Resources**: Truly reusable code lives in `/shared`
5. **Type Safety**: Full TypeScript coverage with strict types

## 📁 Key Directories

- `/features` - Business feature modules
- `/shared` - Shared utilities, hooks, and components
- `/components/ui` - shadcn/ui component library
- `/app` - Next.js pages and routing
- `/i18n` - Internationalization configuration

## 🛠️ Development Workflow

```bash
# Install dependencies
bun install

# Run development server
bun run dev

# Type checking and linting
bun run lint

# Build for production
bun run build
```
