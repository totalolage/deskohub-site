# Task ID: 17
# Title: Implement Pluralization and Variants in Translation System
# Status: done
# Dependencies: 1, 6
# Priority: medium
# Description: Enhance the existing Paraglide JS internationalization system with proper pluralization and variant support, allowing for grammatically correct translations across different languages and contexts by switching to the @inlang/plugin-message-format plugin and implementing MessageFormat 2 syntax.
# Details:
## Objective
Implement proper pluralization and variant support in the translation system by switching from the current @inlang/plugin-json to @inlang/plugin-message-format and adopting MessageFormat 2 syntax.

## Implementation Steps

1. **Update Inlang Configuration**:
   ```javascript
   // project.inlang/settings.json
   {
     // Replace the current JSON plugin with the MessageFormat plugin
     "plugins": [
       "https://cdn.jsdelivr.net/npm/@inlang/plugin-message-format@latest/dist/index.js"
       // Remove @inlang/plugin-json
     ],
     // Other existing configuration...
   }
   ```

2. **Convert Existing JSON Messages to MessageFormat 2 Syntax**:
   ```javascript
   // Example for pluralization in messages
   // Before (JSON format):
   // {
   //   "itemCount": "{{count}} položek"
   // }
   
   // After (MessageFormat 2 syntax):
   {
     "itemCount": [
       "input count",
       "local countPlural = count:plural",
       {
         "cs": {
           "zero": "žádná položka",
           "one": "1 položka",
           "few": "{count} položky",
           "many": "{count} položek",
           "other": "{count} položek"
         },
         "en": {
           "one": "1 item",
           "other": "{count} items"
         }
       }
     ],
     
     "welcomeMessage": [
       "input name, gender",
       {
         "cs": {
           "match": "gender",
           "cases": {
             "male": "Vítejte, pane {name}",
             "female": "Vítejte, paní {name}",
             "other": "Vítejte, {name}"
           }
         },
         "en": {
           "match": "gender",
           "cases": {
             "male": "Welcome, Mr. {name}",
             "female": "Welcome, Ms. {name}",
             "other": "Welcome, {name}"
           }
         }
       }
     ]
   }
   ```

3. **Create Helper Functions for Variant Usage**:
   ```typescript
   // src/utils/i18n-helpers.ts
   import { m } from '@/paraglide/messages';
   
   export function formatItemCount(count: number) {
     return m.itemCount({ count });
   }
   
   export function formatWelcome(name: string, gender: 'male' | 'female' | 'other') {
     return m.welcomeMessage({ name, gender });
   }
   ```

4. **Migrate Existing Translations**:
   - Identify all instances where pluralization is needed (e.g., item counts, days, hours)
   - Convert hardcoded translations to use MessageFormat 2 syntax with declarations and selectors
   - Update component usage to pass the required parameters

5. **Update Component Usage**:
   ```jsx
   // Before
   <p>{m.items} ({itemCount})</p>
   
   // After
   <p>{formatItemCount(itemCount)}</p>
   ```

6. **Document MessageFormat 2 Syntax for Team**:
   - Create a reference guide for the team on how to use the new syntax
   - Include examples for common scenarios (pluralization, gender, formality)
   - Document language-specific pluralization rules
   - Provide links to the official MessageFormat 2 documentation

7. **Handle Complex Pluralization Rules**:
   - Implement Czech-specific pluralization rules (zero, one, few, many, other)
   - Ensure English pluralization follows correct patterns (one, other)
   - Test with various numeric values to verify correct form selection
