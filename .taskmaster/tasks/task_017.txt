# Task ID: 17
# Title: Implement Pluralization and Variants in Translation System
# Status: pending
# Dependencies: 1, 6
# Priority: medium
# Description: Enhance the existing Paraglide JS internationalization system with proper pluralization and variant support, allowing for grammatically correct translations across different languages and contexts.
# Details:
## Objective
Implement proper pluralization and variant support in the translation system following the Paraglide JS variants guide.

## Implementation Steps

1. **Update Paraglide Configuration**:
   ```javascript
   // paraglide.config.js
   export default {
     project: {
       sourceLanguage: "cs",
       supportedLanguages: ["cs", "en"],
       translations: "./src/i18n/translations",
     },
     plugins: [
       // Add any required plugins for variant support
     ],
     // Enable variant support in configuration
     variants: {
       enabled: true
     }
   }
   ```

2. **Implement Variant Syntax in Translation Files**:
   ```javascript
   // Example for pluralization in messages.js
   export const itemCount = {
     cs: "{count, plural, one {1 položka} few {{count} položky} other {{count} položek}}",
     en: "{count, plural, one {1 item} other {{count} items}}"
   }
   
   // Example for gender variants
   export const welcomeMessage = {
     cs: "{gender, select, male {Vítejte, pane {name}} female {Vítejte, paní {name}} other {Vítejte, {name}}}",
     en: "{gender, select, male {Welcome, Mr. {name}} female {Welcome, Ms. {name}} other {Welcome, {name}}}"
   }
   ```

3. **Create Helper Functions for Variant Usage**:
   ```typescript
   // src/utils/i18n-helpers.ts
   import { m } from '@/paraglide/messages';
   
   export function formatItemCount(count: number) {
     return m.itemCount({ count });
   }
   
   export function formatWelcome(name: string, gender: 'male' | 'female' | 'other') {
     return m.welcomeMessage({ name, gender });
   }
   ```

4. **Migrate Existing Translations**:
   - Identify all instances where pluralization is needed (e.g., item counts, days, hours)
   - Convert hardcoded translations to use variant syntax
   - Update component usage to pass the required parameters

5. **Update Component Usage**:
   ```jsx
   // Before
   <p>{m.items} ({itemCount})</p>
   
   // After
   <p>{formatItemCount(itemCount)}</p>
   ```

6. **Document Variant Usage for Team**:
   - Create a reference guide for the team on how to use variants
   - Include examples for common scenarios (pluralization, gender, formality)
   - Document language-specific pluralization rules

7. **Handle Complex Pluralization Rules**:
   - Implement Czech-specific pluralization rules (one, few, many, other)
   - Ensure English pluralization follows correct patterns (one, other)
   - Test with various numeric values to verify correct form selection

# Test Strategy:

2. **Integration Testing with Components**:
